## 1. 目的・課題

### 1.1 背景
肌の状態に合うスキンケアや化粧品を探す作業は、検索・比較・記録が分散して面倒。  
「今日の肌に何をすればいいか」がすぐ分からない。

### 1.2 目的
肌写真を撮るだけで、
1. 肌状態の整理  
2. スキンケア / メイク提案  
3. おすすめ化粧品へのEC誘導  
4. 毎日の成長記録  

までを、1つのWebツールで完結させる。

### 1.3 非目的（やらないこと）
- 医療診断・治療の代替
- 自前ECでの販売・決済
- 高精度な臨床的ニキビ予測

## 3. 機能要件

| ID | 機能 | 詳細 | 状態 |

| F05 | スキンケア提案 | 手順リスト | 実装済 |

| F09 | 画像の即時破棄 | 解析後プレビュー削除 | 実装済 |

## 4. 非機能要件

| 項目 | 要件 |
|---|---|
| 可用性 | AI枠切れでもルールモードで提案可能 |
| セキュリティ | APIキーはサーバーのみ。Origin制限・レート制限 |
| プライバシー | サーバーに写真を永続保存しない。同意必須 |
| パフォーマンス | 画像は最大辺1024px程度に圧縮して送信 |
| デプロイ | GitHub `main` push → Vercel 自動デプロイ |
| 言語 | UI・ユーザー向け文言は日本語 |

## 5. システム構成

```
[ブラウザ]
  BeautyConciergeApp
    → 画像圧縮 / HEIC変換
    → POST /api/analyze
[Vercel / Next.js]
  security（Origin・レート制限・画像検証）
  ai.ts
    → Gemini → OpenAI → rules（無料）
  products.ts（楽天API or 検索リンク）
[外部]
  Google Gemini API / OpenAI API / 楽天市場
```

### 技術スタック
- Next.js 15（App Router）+ TypeScript
- Vercel Hosting
- Gemini / OpenAI（任意）
- jpeg-js（ルールモード用）
- heic2any（クライアントHEIC変換）

## 7. データ設計

### 成長記録（localStorage）
```ts
{
  id: string
  date: string // YYYY-MM-DD
  score: number
  summary: string
  careLevelLabel: string
  dailyTip: string
  concerns: string[]
}
```
※ 写真バイナリは保存しない

### 環境変数
| Key | 必須 | 用途 |
|---|---|---|
| `GEMINI_API_KEY` | 任意 | AI解析 |
| `OPENAI_API_KEY` | 任意 | AI予備 |
| `AI_PROVIDER` | 任意 | `gemini` / `openai` / `rules` |
| `RAKUTEN_APP_ID` | 任意 | 商品API |
| `NEXT_PUBLIC_APP_URL` | 任意 | Origin許可 |

---

## 8. 実装方針

1. **最小構成から**: 静的LP → Next.jsアプリ → セキュリティ → 無料モード  
2. **機能単位でファイル分割**: `ai.ts` / `rules.ts` / `products.ts` / `security.ts` など  
3. **テストしてから次へ**: ルール判定・セキュリティ検証を単体テスト  
4. **日を跨ぐ開発**: 1日ごとの学びを記事シリーズ化

---

## 10. リスクと対策

| リスク | 対策 |
|---|---|
| AI無料枠切れ | ルールモード自動切替 |
| 顔写真の取り扱い | 非保存・同意・解析後破棄 |
| 医療誤認 | 免責表示・断定しない文言 |
| キー漏洩 | Vercel環境変数 / チャットに貼らない |

# 機能案壁打ちメモ — AI美容コンシェルジュ

日付: 2026-07-10  
対象: `my-tool`（公開URL: https://my-tool-bay.vercel.app）

## 1. 解決したい面倒な作業

| 面倒なこと | 今どうしているか | 自動化できる？ |
|---|---|---|
| 肌の調子を見て、何のケアをすべきか調べる | 検索・SNS・店員に聞く | ○ 写真→提案まで自動化 |
| 自分に合いそうな化粧品を探す | 口コミを何ページも見る | ○ キーワード生成→EC誘導 |
| 毎日の肌変化を覚えておく | メモしない / 忘れる | ○ 端末内の日次記録 |
| APIキーやデプロイの手作業 | 手動設定 | △ 一部自動化済み（Vercel連携） |

## 2. API・サービスで実現できること

| 機能 | 使うもの | 実現度 | 備考 |
|---|---|---|---|
| 肌写真の解析・提案 | Gemini / OpenAI Vision | ◎ | 無料枠切れ時あり |
| 枠切れ時の代替提案 | 自前ルール（明るさ・赤み等） | ◎ | 課金なしで継続可 |
| 商品探索・購入導線 | 楽天API / 検索URL | ○ | 自前ECは作らない |
| 公開・自動デプロイ | GitHub + Vercel | ◎ | pushで反映 |
| 認証・クラウド保存 | Auth / DB | △ | MVPでは見送り |
| 医療級の診断 | 専門モデル・医師監修 | × | やらない（リスク大） |

## 3. MVPに入れる機能 / 入れない機能

### 入れる（実装済み〜強化対象）
- 写真アップロード（HEIC対応）
- プライバシー同意
- AI提案（Gemini優先）
- 無料ルールモード（フォールバック）
- スキンケア / メイク提案
- EC誘導（楽天・Amazon検索）
- 毎日の成長記録（localStorage・文章のみ）

### 入れない（意図的に捨てた）
- 自前EC・決済
- 医療診断・搬送レベルの断定
- アカウント必須のクラウド同期
- 店舗在庫のリアルタイム保証

## 4. 壁打ちで決まった方針

1. **「撮る → 提案 → ECへ連れていく → 記録」** までを1本の体験にする  
2. お金がかかるAIは「あると嬉しい」、なくても **ルールモードで止まらない**  
3. 顔写真は保存しない・同意必須・キーはサーバーのみ  
4. 記事では「作ったもの」より **残した/捨てた判断** を書く  

## 5. 次にやること（課題1の流れ）

1. 要件定義書を確定  
2. 不足テストを追加して単体確認  
3. システム・ツール記事を書く（スクショ付き）  
4. 記事URLを提出フォームで報告
